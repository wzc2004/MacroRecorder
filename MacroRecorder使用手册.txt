============================================
  MacroRecorder 使用手册 v1.1
  键盘鼠标宏录制与自动化脚本生成工具
============================================

【工具简介】

MacroRecorder 是一个 Windows 平台的小型自动化工具，它可以：

  - 在后台录制你的键盘和鼠标操作（全局捕获，不限窗口）
  - 自动分析按键模式，生成 PowerShell 自动化脚本
  - 支持长按（Hold）、鼠标点击/拖拽、点击间隔精确回放
  - 鼠标位置以屏幕百分比存储，在不同分辨率下自动适配
  - 将多个宏按播放列表顺序组合成一个智能化脚本
  - 生成可双击运行的 .bat + .ps1 一键自动化方案

适用场景：游戏自动化、重复性操作、跨窗口流程自动化等。


【环境要求】

  - Windows 操作系统
  - Python 3.7+（你的电脑已安装 Python 3.11）
  - pynput 库（双击 install.bat 自动安装）
  - 不需要安装其他第三方软件


【快速开始（5 分钟上手）】

  1. 双击 install.bat → 自动安装依赖 → 启动 MacroRecorder 窗口
  2. 点击 "Start Recording" 按钮
  3. 在 5 秒内切换到你的目标窗口（游戏、软件等）
  4. 执行一遍你想要自动化的完整操作
  5. 按 ScrollLock + F6 停止录制（无需切回工具窗口）
  6. 在弹出的时间轴编辑器中调整时间参数（可选）
  7. 点击 Save，输入宏的名称（留空使用默认名）
  8. 打开输出文件夹（MacroOutputs/宏名称/），双击 run.bat 运行


【停止录制的方法】

  方法一（推荐）：在目标窗口中直接按 ScrollLock + F6
    - 先按住 ScrollLock 不放，再按 F6
    - 不需要切回 MacroRecorder 窗口

  方法二：切回 MacroRecorder 窗口，点击 "Stop Recording" 按钮


【主界面说明】

  顶部状态指示器：
    ● 绿色 Ready      — 就绪，等待开始录制
    ● 红色 Recording  — 正在录制中
    ● 橙色 Processing — 正在处理录制数据
    ● 蓝色 Done       — 处理完成，时间轴编辑器已打开

  按钮区域：
    Start Recording / Stop Recording — 开始/停止录制
    Stop: ScrollLock + F6           — 停止快捷键提示

  鼠标录制复选框：
    Record mouse clicks — 勾选后录制鼠标点击位置和拖拽轨迹
    位置以屏幕百分比（0.0~1.0）存储，自动适配不同分辨率

  输出目录：
    设置自动化脚本的保存位置，默认在 MacroRecorder/MacroOutputs/

  底部按钮：
    Edit Last Recording — 重新打开上一次的时间轴编辑器
    Composer           — 打开宏组合器，将多个宏组合成一个脚本
    Open Output Folder — 在资源管理器中打开输出文件夹


【录制操作详解】

  键盘录制：
    - 所有按键都会被记录：字母、数字、功能键（F1-F12）、回车、
      退格、Tab、方向键、Esc 等
    - 支持标点符号：. , - / ; = [ ] \ ' ` 等
    - 修饰键（Ctrl、Shift、Alt）暂不支持独立录制
    - 组合键（如 Ctrl+C）暂不支持

  长按检测：
    - 按住超过 1 秒的按键自动识别为"长按"
    - 回放时使用 keybd_event 模拟：按下 → 等待 → 释放 → 双保险释放
    - 长按后的操作间隔与录制时完全一致
    - Windows 的自动重复按键事件会被自动过滤（不会生成数百次点按）

  鼠标录制：
    - 点击位置以屏幕百分比存储（例如 0.5, 0.3 = 屏幕正中偏上）
    - 支持左键、右键、中键
    - 拖拽操作自动记录移动轨迹
    - 回放时根据当前屏幕分辨率自动换算绝对坐标
    - 可以点击其他窗口实现跨窗口自动化

  首尾自动裁剪：
    - 录制开始的前 5 秒自动删除（包含你切换窗口的操作）
    - 录制的最后 5 秒自动删除（包含你按停止键的操作）
    - 如果录制时长不足 10 秒，跳过裁剪
    - 裁剪从 timestamps 自动计算，保留真实的按键时间间隔


【时间轴编辑器】

  录制停止后自动弹出，用于查看和调整每个操作的时间参数。

  全局设置（顶部）：
    Key Delay (ms)        — 按键之间的默认延迟（毫秒）
    Cycle Interval (s)    — 每轮循环结束后的等待时间（秒）
    Startup Delay (s)     — 启动后切换窗口的等待时间（秒）
    Loop Count            — 循环次数：0=无限循环，5=执行5轮后停止
    Apply & Rebuild       — 将 Key Delay 应用到所有操作

  操作列表（表格）：
    列名              说明
    ──────────────────────────────────────────
    #                 操作序号
    Action            操作描述（按键名、次数、点击/长按）
    Delay before (ms) 此操作与上一个操作之间的延迟
    Hold Duration (ms)长按的持续时间（仅长按可编辑）

  编辑方法：
    - 双击表格中的任意"Delay"或"Duration"单元格 → 输入新值 → 回车
    - 点击 "Apply & Rebuild" → 将所有操作的延迟统一为 Key Delay 值
    - 点击 "Reset to Auto-Detected" → 恢复为自动分析的值

  底部按钮：
    Reset to Auto-Detected — 恢复默认值
    Cancel                 — 放弃修改
    Save                   — 生成脚本文件


【宏组合器 (Composer)】

  用途：将多个已录制的宏按播放列表顺序组合成一个自动化脚本。

  打开方式：主窗口点击 "Composer" 按钮。

  界面说明：
    左侧 - 已保存的宏：
      自动扫描 MacroOutputs 文件夹中所有包含 recording.json 的宏

    右侧 - 播放列表：
      按顺序列出要执行的宏及其重复次数
      支持拖拽排序、上移/下移按钮
      每次添加时弹出窗口设置重复次数

    底部 - 总循环次数：
      0（默认）= 整个播放列表无限循环
      N = 执行 N 轮后自动停止

  使用流程：
    1. 左侧列表选中一个宏 → 点击 ">> 添加到播放列表 >>"
    2. 输入重复次数（例如 3 次）
    3. 重复以上步骤添加更多宏
    4. 在右侧列表中拖拽调整顺序
    5. 设置总循环次数（可选）
    6. 点击 "生成组合脚本"

  运行提示：
    生成的组合脚本在切换宏时会显示 Write-Host '>> Stage: xxx'
    让你在 PowerShell 窗口看到当前执行到哪个阶段


【生成脚本说明】

  每次保存生成以下文件：
    automation_script.ps1  — 主 PowerShell 自动化脚本
    run.bat                — 双击启动脚本（调用 .ps1）
    使用说明.txt           — 该脚本的中文使用说明
    recording.json         — 宏的完整数据（供组合器使用）

  运行方法：
    1. 进入输出文件夹（MacroOutputs/你的宏名称/）
    2. 双击 run.bat
    3. 看到 PowerShell 窗口，5 秒内切换到目标窗口
    4. 脚本自动开始执行

  停止方法：
    - 切换到 PowerShell 窗口
    - 按 Ctrl + C
    - 如果提示确认，输入 Y 后回车

  自定义修改：
    用记事本打开 automation_script.ps1，修改顶部变量即可：
      $intervalSeconds         — 每轮间隔秒数
      $clickDelayMilliseconds  — 按键间隔毫秒数
      $startupDelaySeconds     — 启动等待秒数
      while ($true)            — 改为 for 循环可控制次数

  循环次数控制：
    - 保存前在时间轴编辑器设置 Loop Count
    - 0 = while ($true)    无限循环
    - 5 = for ($_loop = 0; $_loop -lt 5; $_loop++)  执行 5 次


【录制技巧与最佳实践】

  1. 录制 10 秒以上的操作，确保首尾裁剪后有足够数据
  2. 开始录制后先等 2-3 秒再操作（让裁剪更干净）
  3. 执行完整的一轮操作即可（脚本会自动循环）
  4. 操作结束后等 2-3 秒再按停止（让裁剪更干净）
  5. 多使用时间轴编辑器微调时间参数
  6. 为每个宏起一个有意义的名字（方便组合器识别）
  7. 将复杂流程拆分成小宏，用组合器拼接
  8. 不同操作之间的间隔尽量保持一致


【常见问题】

  Q: 双击 install.bat 提示找不到 python？
  A: 确保 Python 已安装并添加到系统 PATH。
     在命令行运行 python --version 确认。

  Q: 录制后脚本运行的按键不正确？
  A: 确认录制时目标窗口在前台。
     检查是否按了 Ctrl/Shift/Alt（这些键暂不单独支持）。
     标点符号（. , - / 等）已正确映射为虚拟键码。


  Q: 脚本运行时中文显示乱码？
  A: .ps1 文件已使用 UTF-8 BOM 编码保存。
     确保 PowerShell 版本 >= 5.1。

  Q: 如何在不同分辨率的电脑上使用录制的宏？
  A: 鼠标位置使用屏幕百分比存储（如 50% 而不是 960px）。
     回放时自动换算为当前屏幕的绝对坐标。
     键盘操作不受分辨率影响。

  Q: 生成的脚本循环间隔太慢？
  A: 在时间轴编辑器中减小 "Cycle Interval (s)" 的值。
     也可以手动编辑 .ps1 中的 $intervalSeconds 变量。
     默认值为 1 秒。

  Q: 如何让脚本执行指定次数后自动停止？
  A: 保存前在时间轴编辑器设置 Loop Count 为具体数字。

  Q: 能否录制组合键（Ctrl+C、Shift+A 等）？
  A: 当前版本暂不支持。KeySim C# 类已预置 ComboKey 方法，
     未来的版本将支持此功能。

  Q: 保存时输入的名称是否可以用中文？
  A: 可以。中文名称自动作为文件夹名（非法字符会替换为下划线）。

  Q: 如何将几个单独的宏串联起来？
  A: 使用宏组合器（Composer）。每个宏可以设置不同的重复次数。
     生成的组合脚本会按播放列表顺序执行。


【项目文件结构】

  MacroRecorder/
    │
    ├── main.py                主程序（Tkinter 图形界面）
    ├── recorder.py            键盘鼠标录制引擎
    ├── trimmer.py             首尾裁剪模块
    ├── analyzer.py            按键分析与模式检测
    ├── code_generator.py      PowerShell 脚本生成器
    ├── template_engine.py     .bat 和说明文档模板生成
    ├── key_map.py             按键到 VK/SendKeys 映射表
    ├── cs_templates.py        C# 嵌入式代码（KeySim/MouseSim）

    └── 使用说明.txt           本文档
    │
    └── MacroOutputs/          输出文件夹
        ├── 宏名称1/
        │   ├── automation_script.ps1
        │   ├── run.bat
        │   ├── 使用说明.txt
        │   └── recording.json
        └── Composed_xxx/
            └── composed_script.ps1


【注意事项】

  - 运行生成脚本时确保目标窗口在前台
  - 录制时按的首尾 5 秒操作会被自动裁剪
  - 鼠标录制只记录点击和拖拽，不记录空闲移动
  - 鼠标拖拽的中间轨迹会被记录和回放
  - 修饰键（Ctrl/Shift/Alt）暂不单独支持
  - 组合键（如 Ctrl+C）暂不支持，功能已在规划中
